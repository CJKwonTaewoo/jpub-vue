# ToDoリストを作りながら学習しよう！

このチュートリアルでは、書籍の CHAPTER 1～4 で解説しているいろいろな機能を使って、ToDo リストを作成します。 チュートリアルの流れから、Vue.js の基本機能が何を行ってくれるのか？を知ることができるようになっています。

# 完成形

次のような ToDo リストを作成します。

  • ToDo の추가・削除
  • 作業中・完了の作業状態の切りかえ
  • 作業状態ごとの表示の絞り込み
画面のイメージ

ローカルストレージを使っているため、保存した데이터を他の端末から見ることはできません。

# STEP1 인스턴스の作成

まずは、アプリケーションを紐付ける要素 #app を作成します。

index.html
<body>
  <div id="app">
    <!-- ここにテンプレートを書いていく -->
  </div>
</body>

コンストラクタ함수 Vue を使ってルート인스턴스を作成します。 アプリケーションで使用したい데이터は data オプションへ登録していきます。

main.js
const app = new Vue({
  el: '#app',
  data: {
    // 使用する데이터
  },
  methods: {
    // 使用する메소드
  }
})

data オプションへ登録した데이터は、すべてリアクティブ데이터に変換されます。

# STEP2 ローカルストレージ API の使用

데이터はサーバーではなく"ローカルストレージ"へ保存することにします。 ストレージ周りの実装は Vue.js 公式サンプル"TodoMVC の예"のコードを使用させていただきます。

// https://jp.vuejs.org/v2/examples/todomvc.html
var STORAGE_KEY = 'todos-vuejs-demo'
var todoStorage = {
  fetch: function() {
    var todos = JSON.parse(
      localStorage.getItem(STORAGE_KEY) || '[]'
    )
    todos.forEach(function(todo, index) {
      todo.id = index
    })
    todoStorage.uid = todos.length
    return todos
  },
  save: function(todos) {
    localStorage.setItem(STORAGE_KEY, JSON.stringify(todos))
  }
}

この公式のコードの内容에 대해서は詳しく설명しませんが、これは Storage API を使った데이터の取得・保存の처리だけを抜き出したものです。 小さなライブラリだと思ってください。 なぜ、こういった처리の抜き出しをするかは、書籍 CHAPTER7 のコラム(219ページ)で설명しています。 特に手を加える必要はないため、このコードは main.js ファイルの一番上の方に加えておきましょう。

実際にストレージに保存される데이터のフォーマットは、次のような JSON です。

[
  { "id": 1, "comment": "新しいToDo1", "state": 0 },
  { "id": 2, "comment": "新しいToDo2", "state": 0 }
]

# STEP5 リストレンダリング

ToDo リスト데이터用の空の배열を data オプションへ登録します。

これは、데이터が何もない時でも배열として認識されるようにするためと、もともと data オプション直下の데이터は後から추가ができないため初期値で宣言しておく必要があるためです。

var app = new Vue({
  el: '#app',
  data: {
    todos: []
  }
})

テーブルタグの [1] で배열要素の数だけ繰り返し表示させるには、対象となるタグ( ここでは <tr> タグ )に v-for ディレクティブを使用します。

<!-- ここに <tr> で ToDo の要素を1行づつ繰り返し表示 -->
<tr v-for="item in todos" v-bind:key="item.id">
  <!-- 要素の情報 -->
</tr>

ディレクティブの値は JavaScript の式になっており次のように書きます。

v-for="各要素の一時的な이름 in 繰り返したい배열や객체"

v-for を記述したタグとその内側で todos 데이터の各要素のプロパティが使用できるようになります。 <tr> タグの内側に"ID""コメント""状態変更ボタン""削除ボタン"のカラムを추가していきましょう。

<tbody>
  <!-- ここに <tr> で ToDo の要素を1行づつ繰り返し表示 -->
  <tr v-for="item in todos" v-bind:key="item.id">
    <th>{{ item.id }}</th>
    <td>{{ item.comment }}</td>
    <td class="state">
      <!-- 状態変更ボタンのモック -->
      <button>{{ item.state }}</button>
    </td>
    <td class="button">
      <!-- 削除ボタンのモック -->
      <button>削除</button>
    </td>
  </tr>
</tbody>

このボタンはまだなにも機能しないモックのため、機能はこれから実装していきます。

# STEP6 フォーム入力値の取得

新しい ToDo をリストへ추가するための入力フォームを作成します。

ref 属性を使って参照するための이름をタグに付けておくと、その DOM に直接アクセスできます。

<input type="text" ref="comment">

ref 属性で이름を付けたタグは、메소드内から次のように使用できます。

this.$refs.이름

テンプレートでは変数名(プロパティ名)だけで데이터を使用できましたが、메소드内で데이터や메소드を使用するときは this を付ける必要があります。

たとえば、comment の場合なら次のように使用します。

this.$refs.comment.value

実際は、次の STEP7 で使用します。

v-model ディレクティブを使えば데이터とフォーム入力を同期することもできますが、今回は入力した데이터を画面に表示させないのと常に데이터として持っている必要がないため、この $refs を使って入力値を取得することにします。

  <!-- ToDoリストテーブル -->
  </tbody>
</table>

<h2>新しい作業の추가</h2>
<form class="add-form" v-on:submit.prevent="doAdd">
  <!-- コメント入力フォーム -->
  コメント <input type="text" ref="comment">
  <!-- 추가ボタンのモック -->
  <button type="submit">추가</button>
</form>

テーブルの下あたりに추가しておきます。

v-on:submit.prevent="doAdd"

この v-on ディレクティブによって、ボタンを클릭したり入力フォームでエンターを押してフォームのサブミットが行われると、それをハンドリングして doAdd 메소드が呼び出されるようになります。

# STEP7 リストへの추가

つづいて doAdd 메소드を定義しましょう。

この메소드は、フォームの入力値を取得して新しい ToDo の추가처리をします。 ルートコンストラクタの methods オプションに、메소드を登録します。

new Vue({
  // ...
  methods: {
    // ToDo 추가の처리
    doAdd: function(event, value) {
      // ref で이름を付けておいた要素を参照
      var comment = this.$refs.comment
      // 入力がなければ何もしないで return
      if (!comment.value.length) {
        return
      }
      // { 新しいID, コメント, 作業状態 }
      // という객체を現在の todos リストへ push
      // 作業状態"state"はデフォルト"作業中=0"で作成
      this.todos.push({
        id: todoStorage.uid++,
        comment: comment.value,
        state: 0
      })
      // フォーム要素を空にする
      comment.value = ''
    }
  }
})

ちょっと長くなりましたが、$refs を使っている以外は普段使っている JavaScript の構文と同じです。 コメントと一緒に1行づつ読んでみてください。

通常の배열메소드 push を使うだけで、リスト데이터へ추가できます。

# STEP8 ストレージへの保存の自動化

さて、JavaScript 内では데이터は추가されましたが、これではまだローカルストレージに保存されていません。 ブラウザをリロードしたら消えてしまいます。

doAdd 메소드の最後に todoStorage.save 메소드を使って保存してもよいのですが、추가・削除・作業状態の変更すべて同じ처리をしなければいけません。

todos 데이터の内容が変わると、自動的にストレージへ保存してくれたら素敵ですね。 これは watch オプションの"ウォッチャ"機能を使うことで可能です。 ウォッチャは데이터の変化に反応して、あらかじめ登録しておいた처리を自動的に行います。

watch: {
  감시할데이터: function(newVal, oldVal) {
    // 変化した時に行いたい처리      
  }
}

new Vue({
  // ...
  watch: {
    // オプションを使う場合は객체形式にする
    todos: {
      // 引数はウォッチしているプロパティの変更後の値
      handler: function(todos) {
        todoStorage.save(todos)
      },
      // deep オプションでネストしている데이터も監視できる
      deep: true
    }
  }
})

これで、todos 데이터に何か変化があれば自動的にストレージへ保存されるようになりました。

# STEP9 保存されたリストを取得しよう

ストレージへの保存ができたので、次はストレージからの取得です。 このアプリケーションの"인스턴스作成時"に、ローカルストレージに保存されている데이터を"自動的"に取得して、Vue.js の데이터として読み込みましょう。 特定のタイミングに何か처리をはさみたいときは"ライフサイクルフック"の메소드を使用します。

タイミングがいくつか用意されていますが、今回の"인스턴스作成時"には created 메소드を使うとよいでしょう。

new Vue({
  // ...
  created() {
    // 인스턴스作成時に自動的に fetch() する
    this.todos = todoStorage.fetch()
  }
})

데이터の取得には、先に作っておいた todoStorage 객체の fetch 메소드を使用します。

ライフサイクル메소드の定義は"methods の中ではない"ことに注意してください。

ローカルストレージは Ajax と違い同期的に結果を取得できるため、返り値を代入すればいいだけなので簡単です！

# STEP10 状態の変更と削除の처리

つづいて"状態の変更"と"削除"機能を実装しましょう。 methods オプションにそれぞれの메소드を作成します。

# doChangeState 메소드(状態変更)

item.state の値を反転します。

# doRemove 메소드(削除)

インデックスを取得して배열메소드の splice を使って削除します。

どちらも引数として要素の参照を渡しています。

new Vue({
  // ...
  methods: {
    // ...
    // 状態変更の처리
    doChangeState: function(item) {
      item.state = item.state ? 0 : 1
    },
    // 削除の처리
    doRemove: function(item) {
      var index = this.todos.indexOf(item)
      this.todos.splice(index, 1)
    }
  }
})

まだモックの状態だった、状態変更ボタンのイベントをハンドルします。

<button v-on:click="doChangeState(item)">
  {{ item.state }}
</button>

つづいて削除ボタンもハンドルします。 "削除"は注意するべき操作のため、キー修飾子 .ctrl を使って"コントロールキーを押しながら클릭"しなければ呼び出されないようにします。

<button v-on:click.ctrl="doRemove(item)">
  削除
</button>

# STEP11 選択用フォームの作成

特定の作業状態のリストのみを表示させる"絞り込み機能"を추가しましょう。

スローガンテキストの下にラジオボタンをリストで表示します。 ToDo リストと同じように動的に作成するため、選択肢の options リストを作成しました。

data: {
  // ...
  options: [
    { value: -1, label: 'すべて' },
    { value: 0,  label: '作業中' },
    { value: 1,  label: '完了' }
  ],
  // 選択している options の value を記憶するための데이터
  // 初期値を"-1"つまり"すべて"にする
  current: -1
}

options リストを <label> タグで繰り返し描画して、内側の <input> タグの value 属性には、데이터側の label.value 데이터をバインドします。

<label v-for="label in options">
  <input type="radio"
    v-model="current"
    v-bind:value="label.value">{{ label.label }}
</label>

v-model ディレクティブを使って、ラジオボタンの選択値と current 데이터を同期させます。 ラジオボタンが変更されると、その要素の label.value が current プロパティへ代入される仕組みです。

# STEP12 リストの絞り込み機能

current 데이터の選択値によって表示させる ToDo リストの内容を振り分けるため"산출 속성"という機能を使用します。 산출 속성は、데이터から別の新しい데이터を作成する함수型の데이터です。

定義方法は、computed オプションに加工した데이터を返す메소드を登録します。 산출 속성は、元になった데이터に変更があるまで、結果をキャッシュするという性質を持っています。

new Vue({
  // ...
  computed: {
    computedTodos: function() {
      // 데이터 current が -1 ならすべて
      // それ以外なら current と state が一致するものだけに絞り込む
      return this.todos.filter(function(el) {
        return this.current < 0 ? true : this.current === el.state
      }, this)
    }
  }
})

定義方法が違うだけで使い方は데이터と一緒です。 一覧表示テーブルの v-for ディレクティブで使用している todos の部分を computedTodos に置き換えましょう。

変更前
<tr v-for="item in todos" v-bind:key="item.id">
変更後
<tr v-for="item in computedTodos" v-bind:key="item.id">

たとえば"◯件見つかりました"という結果の要素数を表示したいとき、単純にその배열の computedTodos.length を見れば欲しい数字が得られます。

{{ computedTodos.length }} 개의 결과 출력하기中

キャッシュ機能があるおかげで、메소드と違い何度使用しても처리は 1 度しか行われません。

# STEP13 문자열の変換처리

最後の仕上げとして"状態変更ボタン"のラベルが数字になっているのを修正しましょう。

状態変更ボタンで使っている状態の item.state 데이터は、문자열そのものではなく"キー"になる数字を保存しています。 一般的にもカテゴリーなどの데이터では、こういった数字や短い英数字のキーの状態で保存されます。 しかし、このままでは作業中なら"0"完了なら"1"と表示され、まったく意味がわかりません。

絞り込みのセレクトボックス用の options 데이터をもとに、value から label へ変換するための labels 산출 속성を作成します。

computed: {
  labels() {
    return this.options.reduce(function(a, b) {
      return Object.assign(a, { [b.value]: b.label })
    }, {})
    // キーから見つけやすいように、次のように加工した데이터を作成
    // {0: '作業中', 1: '完了', -1: 'すべて'}
  }
}

Mustache で labels 객체を通すように変更します。

<button v-on:click="doChangeState(item)">
  {{ labels[item.state] }}
</button>

これで人が理解できる文字で表示されるようになりました。 このような文字の처리は、フィルタ機能を使っても同じように変換できます。
